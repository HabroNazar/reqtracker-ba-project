# ReqTracker — система аналізу та пріоритизації бізнес-вимог
Навчальний проєкт (колективний проєкт, Етап 4). Консольна інформаційна система для бізнес-аналітика: збір вимог від стейкхолдерів, пріоритизація за методами MoSCoW та Impact/Effort Matrix, аналітика та експорт даних.

Опис

Технічне завдання згенероване за допомогою генеративного ШІ (Claude) та реалізоване мовою Python з використанням ШІ-асистента для кодування.

Повний звіт із технічним завданням, поясненнями та результатами тестування — у файлі Звіт_ТЗ_ReqTracker_BusinessAnalyst.docx.

Структура проєкту

ФайлПризначенняmodels.pyМодель даних "Вимога" (Requirement) та довідники (категорії, MoSCoW, статуси)repository.pyСховище вимог та CRUD-операції (Repository Pattern)analytics.pyАлгоритми пріоритизації (MoSCoW, Impact/Effort) та статистикаreports.pyФормування текстового звіту та експорт у CSVmain.pyКонсольний інтерфейс користувача

Функціональність


Реєстрація нових бізнес-вимог (назва, опис, стейкхолдер, категорія, оцінки впливу/зусиль)
Перегляд та фільтрація вимог за категорією, пріоритетом, статусом
Зміна статусу вимоги протягом життєвого циклу
Автоматична пріоритизація за методом MoSCoW (Must/Should/Could/Won't)
Побудова матриці Impact/Effort (Quick Wins / Major Projects / Fill-ins / Thankless)
Формування аналітичного звіту з текстовими гістограмами
Експорт даних у CSV (для Excel) та TXT


Запуск

Потрібен Python 3.8+, зовнішні бібліотеки не потрібні.

bashpython main.py

Програма одразу завантажує демонстраційний набір із 10 вимог (типовий приклад проєкту впровадження CRM-системи) та відкриває інтерактивне меню.
